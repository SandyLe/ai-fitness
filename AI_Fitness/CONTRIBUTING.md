# 贡献指南

感谢您对AI-Fitness项目的关注！我们欢迎任何形式的贡献，包括但不限于代码、文档、测试和建议。

## 如何贡献

### 报告Bug
- 确保您使用的是最新版本的代码
- 详细描述问题，包括环境信息、复现步骤和预期结果
- 提供相关的日志或截图

### 提交功能请求
- 清晰描述功能需求和使用场景
- 解释该功能如何改善项目
- 如果可能，提供实现思路

### 提交代码

1. Fork项目到您的GitHub账户
2. 创建功能分支：
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. 编写代码并确保遵循项目的代码规范
4. 添加测试用例
5. 确保所有测试通过
6. 提交更改：
   ```bash
   git commit -m "Add some feature"
   ```
7. 推送到您的Fork：
   ```bash
   git push origin feature/your-feature-name
   ```
8. 创建Pull Request

## 代码规范

### Python代码规范
- 遵循PEP 8编码规范
- 使用类型提示
- 编写清晰的文档字符串
- 保持函数和类的单一职责

### 前端代码规范
- 遵循HTML/CSS/JavaScript最佳实践
- 使用语义化的HTML标签
- 编写可维护的CSS类名

## 测试

- 为新功能添加单元测试
- 确保所有现有测试通过
- 测试覆盖率应尽可能高

## 文档

- 更新相关文档
- 为新功能添加使用说明
- 保持文档与代码同步

## 代码审查

所有提交都需要通过代码审查。审查者会关注：
- 代码质量和可维护性
- 功能实现的正确性
- 安全性考虑
- 性能影响

## 行为准则

请遵守我们的行为准则，营造一个友好、包容的社区环境。

## 联系方式
